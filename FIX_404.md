# Fix 404 Error on GitHub Pages

## Quick Checklist

### 1. Check GitHub Pages Configuration
Go to: https://github.com/rafi-khan-cmd/Edmonton-Urban-Growth-Intelligence-Dashboard-Open-Data-/settings/pages

**Must have:**
- ✅ Source: **GitHub Actions** (NOT "Deploy from a branch")
- ✅ If it says "Deploy from a branch", change it to "GitHub Actions" and Save

### 2. Check Workflow Status
Go to: https://github.com/rafi-khan-cmd/Edmonton-Urban-Growth-Intelligence-Dashboard-Open-Data-/actions

**Check:**
- Is there a workflow run? (Should see "Build and Deploy")
- What's the status?
  - 🟡 Yellow = Running
  - ✅ Green = Success
  - ❌ Red = Failed

### 3. If Workflow Failed
Click on the failed workflow and check which step failed:
- "Fetch data from APIs" - API issues (workflow continues)
- "Run pipeline" - Data or code issues
- "Upload artifact" - File generation issues
- "Deploy to GitHub Pages" - Deployment issues

### 4. If Workflow Succeeded But Still 404
1. Wait 2-3 minutes (Pages can take time to update)
2. Hard refresh the page (Ctrl+F5 or Cmd+Shift+R)
3. Check Pages settings again - should show "Your site is live at..."

### 5. Manual Fix: Create a Simple Test Deployment

If the pipeline is failing, we can deploy a simple version first to test Pages:

1. The workflow should work, but if it keeps failing, check the error logs
2. Make sure `site/index.html` exists (it does)
3. The workflow uploads `./site` directory which contains index.html

## Most Common Issues

### Issue 1: Pages Source Not Set to "GitHub Actions"
**Fix:** Go to Settings → Pages → Source → Select "GitHub Actions" → Save

### Issue 2: Workflow Never Ran
**Fix:** 
- Check if you pushed to `main` branch
- Go to Actions tab and click "Run workflow" manually

### Issue 3: Workflow Failed Before Deployment
**Fix:**
- Check the error in Actions tab
- Fix the error and push again
- Or temporarily skip the pipeline and just deploy the static site

### Issue 4: Artifacts Not Generated
**Fix:**
- The pipeline must complete successfully
- Check that `site/assets/` has files after pipeline runs
- If pipeline fails, artifacts won't exist

## Test Deployment (Skip Pipeline)

If you want to test Pages deployment without running the pipeline, you can temporarily modify the workflow to just deploy the static site. But first, let's see what the actual error is.
